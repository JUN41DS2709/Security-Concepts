# AAA (Authentication, Authorization, and Accounting)



**AAA (Authentication, Authorization, and Accounting)\*\* is a fundamental cybersecurity framework used to control, monitor, and improve the security of devices, systems, networks, and data. It ensures that only authorized users can access resources, defines what actions they are allowed to perform, and records their activities for auditing and accountability.



**Note:** The official term is \*\*Accounting\*\*, but it is often explained as \*\*Accountability\*\* because it keeps users accountable for their actions.



---



# AAA STANDS FOR



- Authentication

- Authorization

- Accountability (Officially called **Accounting**)



---



# Authentication



Authentication verifies the user's identity, whether the user really exists or not. It's like identifying the user.

It can be done using:

- Usernames / Passwords
  
- MFA (Multi-Factor Authentication)

- Biometrics

- Smart Cards

- Security Tokens

etc.

---



# Authorization
Authorization verifies what permissions or access level the user has, what tasks the user is authorized to perform, and what tasks the user is restricted from performing. So basically, it verifies \*\*what permissions do you have?\*\*

It can be:



- **Guest User** (least permissions, just to hang around and look for things, can't perform important tasks)

- **Normal User** (medium permissions required to perform day-to-day tasks)

- **Admin / Root** (highest permission where you own the system. You have higher privileges and can change, edit, delete, or do whatever you are authorized to do.)

---

# Accountability (Accounting)

Accountability is like keeping track of the user's activity.



- User login → **Authentication**

- User permissions → **Authorization**
 
- Keeping track of user activity → **Accountability (Accounting)**



So, accountability is like monitoring what the user is doing after logging in, what files they are accessing, what changes they are making, and what resources they are using.



Examples include:



- Logs

- Records

- Resources accessed

- Session duration

- Login history

- Actions performed


These records help with auditing, troubleshooting, forensic investigations, and making users accountable for their actions.



---



# Key Points



- **Authentication** → Who are you?

- **Authorization** → What are you allowed to do?

- **Accountability (Accounting)** → What are you doing? / What did you do?

### Easy Way to Remember



1. **Authentication** → Verify Identity

2. **Authorization** → Verify Permissions

3. **Accountability (Accounting)** → Track User Activity
